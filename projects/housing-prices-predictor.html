Suppose you suddenly find yourself in financial hardship and you are desperately trying to sell your property. You know that if you did that through a third party (aka real estate agency), you would need to share a proportion of the profit from the sale with them.

Further suppose it's now 1978 and that you live in the suburbs of Boston and you have access to the following info about housing prices in your area.

With your knowledge in machine learning, you can predict the price at which you can sell your house such that the profit is maximized.

Developing Models ¶

Baseline model¶

A naive model would be to predict the price of any house to be the average (mean, in statistics speak) price in the area regardless of the features of the house. This is obviously not a good model as it gives the same value to a house with 5 rooms and a house with 1 room.

However, it gives us a baseline to evaluate other models against. If the models we come up with perform worse than this naive model, then, we know that there must be some bug in our implementation!

Machine Learning Abstraction ¶

To perform linear regression (aka fitting curve to data points), we need to define the following:

  1. Hypothesis This is our function that takes in the number of rooms (feature) and outputs the prediction for the price of the property:

    $$h_\theta(x) = \Theta_0 + \Theta_1*x_1 $$

  2. Cost Function We need a way to evaluate the performance of our algorithm. For linear regression, we can use something called the mean squared error loss. Essentially, we compute squared of the error the algorithm makes.

    In our dataset, error is defined as the difference between the actual housing price and the predicted housing price:

    $$error = prediction - actual$$

    $$error = h_\theta(x) - y$$

  3. Optimization function If you will, the machine learning is basically to learn the Theta, aka the model. To do that, we can use gradient descent. The update rule is the following:

    $$\Theta = \Theta - \alpha \frac{1}{m} \Sigma_{i = 1}^{m} (h_\theta(x^{(i)})-y^{(i)})x^{(i)}$$

where $\alpha$ is called learning rate, also known as the step size. It should be chosen empirically because the best choice depends upon the input problem.

If the learning rate is too small, it will take a long time for the algorithm to converge.

If the learning rate is too big, it might take a long time to converge as well, but for a different reason: it jumps right over the region with zero risk and oscillates back and forth for a long time.

Now, we are ready to translate the above to code!

def predict(X, Theta):
    return X.dot(Theta)

def cost(X, Y, Theta):
    return np.sum((predict(X, Theta) - Y)**2)/(2*len(X))
    
def descend(X, Y, Theta, alpha):
    return Theta - alpha*(np.mean(predict(X, Y, Theta) - Y)**2)/len(X)

Linear Regression as Python Class ¶

To make the code cleaner, we can also implement the machine learning algorithm in a Python Class, thus abstracting certain things from the user, which is a common software engineering practice.

In [14]:
class LinearRegression(object):
    
    def __init__(self, alpha=0.002, num_iter=50):
        self.alpha = alpha
        self.num_iter = 50
        self.costs = []
        
    def fit(self, X, y):
        self.model = np.zeros(1 + X.shape[1])
        
        for i in range(self.num_iter):
            prediction = self.predict(X)
            errors = y - prediction
            self.model[1:] += self.alpha * X.T.dot(errors)
            self.model[0] += self.alpha * np.sum(errors)
            cost = np.sum(np.power(errors, 2))/len(X)
            self.costs.append(cost)
        
        return self.costs
        
    def predict(self, X):
        return np.dot(X, self.model[1:]) + self.model[0]
