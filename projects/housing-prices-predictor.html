Split Data Into Train and Validation Sets ¶

Here, we are taking the Boston housing dataset and splitting it intor training and validation sets. Data is also shuffled into random order to remove any bias that may be inherent in the data.

Astute readers may be wondering: what is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?

The short answers to the question above are as follow:

  1. Without splitting there is no way to test the effectiveness of our model, unless we have additional data.
  2. We want our model to generalize, as in, predicting the price of a house it hasn't seen before (in our dataset) as accurately as possible. If we don't have a way to test our model, our model might be overfitting the data in our training set, hence, it might end up predicting the price of a new house very inaccurately.

The code below splits the 80% of the original data into the training set and 20% into the validation set.

Developing Models ¶

Baseline model ¶

A naive model would be to predict the price of any house to be the average (mean, in statistics speak) price in the area regardless of the features of the house. This is obviously not a good model as it gives the same value to a house with 5 rooms and a house with 1 room.

However, it gives us a baseline to evaluate other models against. If the models we come up with perform worse than this naive model, then, we know that there must be some bug in our implementation!

In [59]:
class NaiveModel:
    
    def fit(self, X_train, y_train):
        self.pred = np.mean(y_train)
        return self
    
    def predict(self, X):
        preds = np.array([self.pred for _ in range(X.shape[0])])
        return preds

Machine Learning Abstraction ¶

To perform linear regression (aka fitting curve to data points), we need to define the following:

  1. Hypothesis This is our function that takes in the number of rooms (feature) and outputs the prediction for the price of the property:

    $$h_\theta(x) = \Theta_0 + \Theta_1*x_1 $$

  2. Cost Function We need a way to evaluate the performance of our algorithm. For linear regression, we can use something called the mean squared error loss. Essentially, we compute squared of the error the algorithm makes.

    In our dataset, error is defined as the difference between the actual housing price and the predicted housing price:

    $$error = prediction - actual$$

    $$error = h_\theta(x) - y$$

  3. Optimization function If you will, the machine learning is basically to learn the Theta, aka the model. To do that, we can use gradient descent. The update rule is the following:

    $$\Theta = \Theta - \alpha \frac{1}{m} \Sigma_{i = 1}^{m} (h_\theta(x^{(i)})-y^{(i)})x^{(i)}$$

where $\alpha$ is called learning rate, also known as the step size. It should be chosen empirically because the best choice depends upon the input problem.

If the learning rate is too small, it will take a long time for the algorithm to converge.

If the learning rate is too big, it might take a long time to converge as well, but for a different reason: it jumps right over the region with zero risk and oscillates back and forth for a long time.

Now, we are ready to translate the above to code!

def predict(X, Theta):
    return X.dot(Theta)

def cost(X, Y, Theta):
    return np.sum((predict(X, Theta) - Y)**2)/(2*len(X))
    
def descend(X, Y, Theta, alpha):
    return Theta - alpha*(np.mean(predict(X, Y, Theta) - Y)**2)/len(X)

Linear Regression as Python Class ¶

To make the code cleaner, we can also implement the machine learning algorithm in a Python Class, thus abstracting certain things from the user, which is a common software engineering practice.

In [65]:
class LinearRegressionGD(object):
    
    def __init__(self, alpha=0.002, num_iter=50):
        self.alpha = alpha
        self.num_iter = 50
        self.costs = []
        
    def fit(self, X, y):
        self.model = np.zeros(1 + X.shape[1])
        
        for i in range(self.num_iter):
            prediction = self.predict(X)
            errors = y - prediction
            self.model[1:] += self.alpha * X.T.dot(errors)
            self.model[0] += self.alpha * np.sum(errors)
            cost = np.sum(np.power(errors, 2))/len(X)
            self.costs.append(cost)
        
        return self.costs
        
    def predict(self, X):
        return np.dot(X, self.model[1:]) + self.model[0]

Analyzing Performances ¶

Coefficient of Determination, R^2 ¶

We are now going to train our 3 models (naive, linear regression, decision tree) on the training set with all housing features and then compute the $R^2$ score for each of them.

$R^2$ is known as the coefficient of determination. It is a useful statistic in describing how good a model is at making predictions.

The values for $R^2$ range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an $R^2$ of 0 always fails to predict the target variable, whereas a model with an $R^2$ of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features.

from sklearn.metrics import r2_score

def eval_model(model):
    """ Calculates and returns the performance score between 
    true and predicted values based on the metric chosen. """
    model.fit(X_train, y_train)
    train_preds = model.predict(X_train)
    print("Train accuracy: " + str(r2_score(y_train, train_preds)))
    test_preds = model.predict(X_test)
    print("Test accuracy: " + str(r2_score(y_test, test_preds)) + '\n')

Learning Curves and Complexity Curves ¶

In Data Science, visualizations are often used in helping us make decisions.

Below, we are plotting the learning and complexity curves.

They help us get an idea of how well the model can generalize to new data. In the plots below, both learning curves converges to 0.78 when max_depth = 3. When max_depth = 10, our Decision Tree is overfitting, thus, it performs well on the training set and relatively poorly on unseen data (test set).

The only difference between the two is that learning curves measure a model's performance on experience (if you have a higher number of training points, you have more experience) while complexity curves measure a model's performance based on the complexity of the model (in decision trees, models become more complex as depth increases).

Bias-Variance Tradeoffs ¶

max_depth = 1: high bias, because both the training and test scores are low, hence, model is underfitting.

max_depth = 10: high variance, because the training score is much higher than the test score, hence, model is overfitting.

Grid Search ¶

Grid search is a way of performing hyperparameter optimization by exhaustively searching through a manually specified subset of the hyperparameter space. Its performance is measured on the cross-validation set. The technique chooses the value of the hyperparameter that performs best on the cross-validation set.

Here, we are going to use grid search to optimize the 'max_depth' parameter for our decision tree model.

Applicability ¶

Sensitivity¶

An optimal model is not necessarily a robust model. Sometimes, a model is either too complex or too simple to sufficiently generalize to new data. Sometimes, a model could use a learning algorithm that is not appropriate for the structure of the data given. Other times, the data itself could be too noisy or contain too few samples to allow a model to adequately capture the target variable — i.e., the model is underfitted. Run the code cell below to run the fit_model function ten times with different training and testing sets to see how the prediction for a specific client changes with the data it's trained on.

def predictTrials(X, y, fitter, data):
    """ Performs trials of fitting and predicting data. """

    # Store the predicted prices
    prices = []

    for k in range(10):
        # Split the data
        X_train, X_test, y_train, y_test = train_test_split(X, y, \
            test_size = 0.2, random_state = k)
        
        # Fit the data
        reg = fitter(X_train, y_train)
        
        # Make a prediction
        pred = reg.predict([data[0]])[0]
        prices.append(pred)
        
        # Result
        print "Trial {}: ${:,.2f}".format(k+1, pred)

    # Display price range
    print "\nRange in prices: ${:,.2f}".format(max(prices) - min(prices))
