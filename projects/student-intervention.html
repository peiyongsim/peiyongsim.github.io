Introduction ¶

Background Story¶

Imagine that you are an educator at a public high school in Portugal, where the pupil-teacher ratio is very high. You care about your students and genuinely want them to succeed.

From exam scores, you have a vague idea of which students are excelling and which students are struggling. However, with the limited time and resources you have, it is impossible to give every student the attention they need.

Fortunately, with your experience in Machine Learning and this dataset about your students, you can determine which students are at the highest risks of failing. Then, you can intervene (by talking to them, giving extra review sessions etc), thereby increasing their chances of succeeding!

In this project, we can build a model that classifies students into two categories those who are at risk of failing (y = 1), and those who are not (y = 0).

Furthermore, we can obtain the conditional probability $P(y=1|\ student\ profile)$ and approach the ones with the highest chances of failing.

Thus, in the section that follows, we are going to preprocess our data so that we can extract as much useful information as possible.

Data Cleaning ¶

Handling Categorical Data ¶

There are two types of categorical features--ordinal and nominal. The former are values that can be ordered. One such example is shirt sizes (L > M and M > S) On the other hand, the latter are values that do not have any intrinsic ordering, such as shirt color.

The features in our dataset are all nominal.

Fortunately, many of them are simply yes/no, e.g. internet. These can be reasonably converted into 1/0 (binary) values.

Other columns, like Mjob and Fjob, have more than two values, and are known as categorical variables. The recommended way to handle such a column is to create as many columns as possible values (e.g. Fjob_teacher, Fjob_other, Fjob_services, etc.), and assign a 1 to one of them and 0 to all others.

In [5]:
def preprocess_features(X):
    ''' Preprocesses the student data and converts non-numeric binary variables into
        binary (0/1) variables. Converts categorical variables into dummy variables. '''
    output = pd.DataFrame(index = X.index)
    for col, col_data in X.iteritems():
        if col_data.dtype == object:
            col_data = col_data.replace(['yes', 'no'], [1, 0])
        if col_data.dtype == object:
            col_data = pd.get_dummies(col_data, prefix = col)  
        output = output.join(col_data)
    return output

Developing Models ¶

Naive/Baseline model ¶

A naive model would be to always predict a student would fail XOR succeed, depending on which is higher in the training set.

It makes no sense but it still gives us a baseline to evaluate other models against. If the models we come up with perform worse than this naive model, then, we know that there must be some bug in our implementation!

class NaiveModel:
    
    def fit(self, X_train, y_train):
        self.pred = np.sum(y_train) > len(y_train)
        return self
    
    def predict(self, X):
        preds = np.array([self.pred for _ in range(X.shape[0])])
        return preds

An Aside: ¶

There are 3 different ways to build classifiers.

The first way is by using generative models which assumes sample points originate from some probability distributions. One such example is linear discriminant analysis (LDA).

The second way is by using discriminative models, where we model the posterior probability $P(Y|X)$ directly. This is useful when we care about the confidence of our prediction. Besides that, probabilities provide more granular information. One application is sentiment analysis.

The third and last way is by finding decision boundary directlymuch like how humans' reasoning by analogy. This method does not utilize posterior probability. One class of algorithms that does this is the kernel methods (e.g. SVM)

Implementing Logistic Regression from Scratch ¶

Defining the sigmoid function

The sigmoid function ensures that any real number $\mathbb{R}$ is mapped to a number from 0 to 1.

In [9]:
def sigmoid(z):
    return 1/(1 + np.exp(-z))

Regularized Cost function

$l(\beta) = \lambda\|\beta\|^2_2 -\sum_{i=1}^n \left[y_i\log\mu_i + (1-y_i)\log(1-\mu_i)\right]$

where $\mu_i = 1/(1+\exp(-\beta x_i))$, and $\lambda > 0$ is the regularization parameter.

In [10]:
# translate to code
def regCostFunction(w, X, y, reg):
    n = X.shape[0]
    h = sigmoid(X.dot(w))
    J = (reg/n)*np.power(np.linalg.norm(w), 2) - (1/n)*(np.log(h).T.dot(y) + np.log(1-h).T.dot(1-y))
    return J[0]

Optimization Algorithm ¶

After identifying our application (classification) and selecting our model (logistic regression), our problem now turns into an optimization problem.

Optimization problems can be divided into unconstrained and constrained problems. We solve the former by finding a $\beta$ that minimizes (or maximizes) a continuous objective function $l(\beta)$.

If the function is convex, then we are guaranteed to find a global minimum (or maximum). Otherwise, we can only find the local optima.

Subsequently, we need to decide on the optimization algorithm (e.g. gradient descent, simplex, SVD).

For logistic regression, we will implement both gradient descent (GD) and stochastic gradient descent (SGD).

In [11]:
def gradientDescent(X, y, reg=0.1, alpha=0.1, num_iters=1000):
    n = X.shape[0]
    w = np.zeros((X.shape[1], 1))
    costs = []
    errors = []
    for i in range(1, num_iters):
        rate = alpha/(1 + alpha*reg*i)
        h = sigmoid(X.dot(w))
        grad = (2*reg*w - X.T.dot(h-y))/n
        w += rate*grad
        costs.append(regCostFunction(w, X, y, reg))
    return w, costs, errors

def sgd(X, y, reg=0.1, alpha=0.1, num_iters=10000):
    w = np.zeros((X.shape[1], 1))
    costs = []
    n = X.shape[0]
    
    for t in range(1, num_iters):
        rate = alpha/(1 + alpha*reg*t)
        i = np.random.randint(0, X.shape[0] - 1)
        yi = y[i][0]
        xi = np.reshape(X[i], (w.shape[0], 1))
        h = xi.T.dot(w)
        grad = (2*reg*w + xi*(yi-h))/n
        w = w + rate*grad
        costs.append(regCostFunction(w, X, y, reg))
    return w, costs

Making Predictions with Logistic Regression¶

We are trying to make 0-1 classfication. However, the output of our logistic regression model is a real number $R$.

Thus, we need to somehow convert the real number into either 0 or 1. The 'predict' function below does just that by converting values greater than or equal to threshold to '1' and any other values to '0'.

The threshold should be picked by analyzing the ROC Curves.

def predict(w, X, threshold=0.5):

from sklearn.metrics import accuracy_score

class LogisticRegressionModel:

    def __init__(self, debug=False):
        # if debug True, then plot cost function
        self.debug = debug
        
    def sigmoid(self, z):
        return 1/(1 + np.exp(-z))
        
    def regCostFunction(self):
        n = self.X_train.shape[0]
        h = sigmoid(self.X_train.dot(self.w))
        J = (self.reg/n)*np.power(np.linalg.norm(self.w), 2) - (1/n)*(np.log(h).T.dot(self.y_train) + np.log(1-h).T.dot(1-self.y_train))
        return J[0]
    
    
    def gradientDescent(self):
        n = self.X_train.shape[0]
        self.w = np.zeros((self.X_train.shape[1], 1))
        self.costs = []
        for i in range(1, self.num_iters):
            rate = self.alpha/(1 + self.alpha*self.reg*i)
            h = self.sigmoid(self.X_train.dot(self.w))
            grad = (2*self.reg*self.w - self.X_train.T.dot(h-self.y_train))/n
            self.w += rate*grad
            if self.debug:
                self.costs.append(self.regCostFunction())
        
    def fit(self, X_train, y_train, reg=0.01, alpha=0.5, num_iters=10000, threshold=0.5):
        self.X_train = X_train
        self.y_train = y_train
        self.reg = reg
        self.alpha = alpha
        self.num_iters = num_iters
        self.threshold = threshold
        self.gradientDescent()
        if self.debug:
            print(self.costs)
    
    def predict_probs(self, X):
        return X.dot(self.w)
    
    def predict(self, X):
        return self.predict_probs(X) >= self.threshold
        
    def predict_acc(self, X, y):
        preds = self.predict(X)
        return accuracy_score(y, preds)

Implementing k-Nearest Neighbors ¶

The idea behind kNN classification is very simple. Given a new sample point, you look at the 'k' 'nearest' sample points in the training set and assign the majority class to that point.

'Nearest' is based on some distance metric such as Euclidean distance or Manhattan distance.

'k' varies per application. We can find the optimal value by grid search.

Advantages of kNN: kNN is simple and there's a theorem that says that it works well for large datasets.

Disadvantages of kNN: The computational complexity of classifying a new data point grows linearly with the number of samples in the training set. This is because kNN has no ~training step~.

Here, we are using the Euclidean distance as our metric:

$$distance = \sqrt{(x_i-x_j)^2 + (y_i-y_j)^2 + (z_i-z_j)^2 + ...} $$

where the x, y, z etc with subscripts 'i' denote the features of a sample point $i \neq j$ in the data set.

In [24]:
class kNearestNeighborsModel:
    
    def fit(self, X_train, y_train, k=9):
        self.X_train = X_train
        self.y_train = y_train
        self.k = k
        
    def distance(self, pt1, pt2):
        return np.sqrt(np.sum(np.power(pt1 - pt2, 2)))
    
    def majority_vote(self, indices):
        return np.sum([self.y_train[i] for i in indices]) > len(indices)/2
        
    def predict(self, X):
        preds = []
        for pt1 in X:
            dists = np.array([self.distance(pt1, pt2) for pt2 in self.X_train])
            closest_indices = dists.argsort()[:self.k]
            preds.append(self.majority_vote(closest_indices))
        return preds

Choosing the Best Model ¶

Model Performance Metrics ¶

As mentioned earlier, because the positive classes are imbalance (67% and 33%, respectively), we should be measuring the relevance of our model using a metric called the F-1 score, which is defined as follows:

$$F1 = 2*\frac{PRE*REC}{PRE + REC}$$

To understand F1, we first need to understand the some terminologies, which is best illustrated in the classical tumor diagnostic test. In the tumor diagnostic setting, 'positive' means tumor is malignant while 'negative' means tumor is benign.

  1. True Positive (TP): Model predicts tumor is malignant and tumor is actually malignant
  2. False Positive (FP): Model predicts tumor is malignant but tumor is actually benign
  3. True Negative (TN): Model predicts tumor is benign and tumor is actually benign
  4. False Negative (FN): Model predicts tumor is benign but tumor is actually malignant

This is best captured by the confusion matrix below:

$PRE$ denotes precision and is defined as follows:

$$PRE = \frac{TP}{TP + FP}$$

$REC$ denotes recall and is defined as follows:

$$REC = \frac{TP}{FN + FP}$$

Here are the intuitions behind precision and recall, based on our running example of tumor diagnostic:

Precision: Of all tumors where we predicted y = 1 (malignant), what fraction is actually malignant?

Recall: Of all tumors that are actually malignant, what fraction did we correctly predict as being malignant?

Aside: precision and recall give score of 0 to a "non-learning" algorithm that "predicts" 0 or 1 all the time.

F1 Score:

After understanding the terminologies, and are convinced that they are useful metrics for dataset with skewed classes, we now have to somehow relate the two metrics and decide on the optimal model.

Idea 1: We can compute the average $\frac{PRE+REC}{2}$ but it is prone to cases where $PRE=0.02$ and $REC=1.0$.

Idea 2: $F1 = 2*\frac{PRE*REC}{PRE + REC}$

Analyses and Model Selection ¶

Observations:

  1. kNN has no training step, as mentioned above.
  2. Logistic regression is computationally most feasible; the prediction time is almost 0 because it's just a matrix multiplication, which has been optimized by numpy.

I have decided to select SVM (with Gaussian kernel) as the final model to optimize for the following reasons:

  1. Though kNN has slightly higher F1 score than SVM, kNN might be an issue if the sample size becomes higher.

However, we should use logistic regression if we care about the probabilities (here, logistic regression performs worse than SVM, that's why I'm going with SVM).
